validate()

Mesma assinatura de normalize(), mas nunca lança. Retorna sempre um objeto { valid, value, error } — ideal quando você precisa do resultado booleano sem envolver o código em try/catch.

Assinatura #

Tipo TypeScript 
function validate({
  type:    string,
  value:   unknown,
  options?: object,
}): { valid: boolean; value: string | null; error: string | null }

Objeto de retorno #

Propriedade Quando valid: true Quando valid: false
valid true false
value String normalizada e pronta para uso null
error null Mensagem descritiva do handler

As três propriedades são sempre presentes — você nunca precisa checar se value existe antes de usar. O contrato é claro: se valid é true, value tem o dado normalizado; se valid é false, error tem a mensagem.

Exemplos #

validate() — retorno nos dois casos 
import { validate } from 'data-handlers'

validate({ type: 'cpf', value: '11144477735' })
// → { valid: true, value: '111.444.777-35', error: null }

validate({ type: 'cpf', value: '000.000.000-00' })
// → { valid: false, value: null, error: '[normalize:cpf] Invalid CPF...' }

validate({ type: 'name', value: '' })
// → { valid: false, value: null, error: '[normalize:name] Expected non-empty string' }

Uso típico #

O padrão mais comum com validate() é a validação condicional — checar se o dado é válido e agir diferente em cada caso, sem precisar capturar exceção. O value já vem normalizado quando válido, não é necessário chamar normalize() depois.

Validação e uso do valor normalizado 
const result = validate({ type: 'cpf', value: inputDoUsuario })

if (!result.valid) {
  mostrarErro(result.error)
} else {
  salvarNoBanco(result.value) // '111.444.777-35' — já normalizado
}
Desestruturação direta 
const { valid, value, error } = validate({ type: 'email', value: form.email })

setFieldValid('email', valid)
setFieldError('email', error)
if (valid) setNormalizedValue('email', value)

Com opções do handler #

As options são repassadas ao handler da mesma forma que em normalize(). Se o valor passar, o value retornado já reflete as opções aplicadas.

validate() com options 
// Valida um número e já formata como moeda
validate({ type: 'number', value: 9.9, options: { style: 'currency', currency: 'BRL' } })
// → { valid: true, value: 'R$ 9,90', error: null }

// UUID exigindo versão 4
validate({ type: 'uuid', value: someId, options: { version: 4 } })
// → { valid: true/false, ... }

validate() vs normalize() #

Situação Usar
Salvar no banco, processar dado confiável normalize() — lança e interrompe se inválido
Validar input do usuário, dar feedback validate() — nunca lança, retorna o resultado
Validar múltiplos campos de um objeto schema() — processa tudo e retorna todos os erros
Tipo dinâmico (vem de variável) normalize() ou validate() — ambos aceitam string
Alias no proxy: handlers.cpf.safe(v) é exatamente equivalente a validate({ type: 'cpf', value: v }). Use o que deixar o código mais legível — o resultado é idêntico.
