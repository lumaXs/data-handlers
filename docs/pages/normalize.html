normalize()

Função de nível superior para normalização direta. Recebe um objeto com type, value e options opcionais. Lança um TypeError se o valor for inválido — use quando quiser falhar ruidosamente e travar o fluxo.

Assinatura #

Tipo TypeScript 
function normalize({
  type:    string,
  value:   unknown,
  options?: object,
}): string

O parâmetro type é case-insensitive e ignora espaços em volta — 'name', 'Name' e ' NAME ' são equivalentes. O value pode ser qualquer coisa; cada handler define o que considera válido e lança com uma mensagem descritiva se não passar.

Exemplos #

normalize() — uso geral 
import { normalize } from 'data-handlers'

normalize({ type: 'name',   value: '  joao   silva  ' })
// → 'Joao Silva'

normalize({ type: 'cpf',    value: '11144477735' })
// → '111.444.777-35'

normalize({ type: 'number', value: 1234567.89, options: { style: 'currency', currency: 'BRL', locale: 'pt-BR' } })
// → 'R$ 1.234.567,89'

normalize({ type: 'date',   value: new Date(), options: { dateStyle: 'long', locale: 'pt-BR' } })
// → '3 de março de 2026'

Comportamento com erros #

Quando o valor não passa na validação do handler, normalize() lança um TypeError com uma mensagem no formato [normalize:<tipo>] <motivo>. Isso facilita identificar exatamente qual campo e qual tipo causou o problema.

Exemplos de erros lançados 
// CPF inválido — dígito verificador errado
normalize({ type: 'cpf', value: '000.000.000-00' })
// → throws TypeError: [normalize:cpf] Invalid CPF. Received: 000.000.000-00

// Tipo incorreto para number
normalize({ type: 'number', value: '123' })
// → throws TypeError: [normalize:number] Expected number, received string

// String vazia para name
normalize({ type: 'name', value: '' })
// → throws TypeError: [normalize:name] Expected non-empty string
Capturar o erro: se você quer tratar o caso de erro sem deixar o processo explodir, envolva em try/catch ou use validate() que nunca lança. Para validar objetos inteiros com múltiplos campos, prefira o sistema de schema().
Capturando o TypeError manualmente 
try {
  const cpf = normalize({ type: 'cpf', value: inputDoUsuario })
  salvarNoBanco(cpf)
} catch (err) {
  if (err instanceof TypeError) {
    console.error(err.message) // '[normalize:cpf] Invalid CPF...'
  }
}

normalize() vs handlers proxy #

normalize() é a função de baixo nível — você passa o tipo como string. O objeto handlers é um Proxy que oferece a mesma funcionalidade com uma sintaxe mais fluente. Ambos chamam o mesmo handler internamente; escolha o que deixar o código mais legível no seu contexto.

As duas formas são equivalentes 
import { normalize, handlers } from 'data-handlers'

// Forma explícita com normalize()
normalize({ type: 'cpf', value: '11144477735' })
// → '111.444.777-35'

// Forma fluente com handlers proxy
handlers.cpf.normalize('11144477735')
// → '111.444.777-35'

// Útil quando o tipo é dinâmico (vem de uma variável)
const type = obterTipoDoCampo(campo)
normalize({ type, value: campo.valor })  // não tem como usar o proxy aqui
Quando preferir normalize(): quando o tipo é dinâmico — vem de uma configuração, de um mapa de campos, ou é calculado em runtime. Nesses casos o proxy handlers[type] funciona também, mas normalize() é mais explícito sobre a intenção.
