Plugins nacionais

Plugins para documentos, endereços e formatos brasileiros. Todos seguem o mesmo contrato dos handlers do core — .normalize() lança se inválido, .safe() nunca lança. Todos são idempotentes: passar um valor já formatado retorna o mesmo valor.

Referência viva: a lista completa de tipos registrados está sempre disponível em runtime via handlers.types. Novos plugins podem ser adicionados além dos documentados aqui.

Plugin: cpf #

Valida usando o algoritmo oficial de dígito verificador (dois passes) e formata no padrão 000.000.000-00. Rejeita sequências repetidas como 111.111.111-11 mesmo que os dígitos verificadores batam — esses CPFs são matematicamente válidos, mas não são atribuídos pela Receita Federal.

cpf pt-BR 
// Formata a partir de dígitos puros
handlers.cpf.normalize('11144477735')
// → '111.444.777-35'

// Idempotente — CPF já formatado passa sem alteração
handlers.cpf.normalize('111.444.777-35')
// → '111.444.777-35'

// Sequência repetida é rejeitada (não atribuída pela Receita Federal)
handlers.cpf.safe('00000000000')
// → { valid: false, error: '[normalize:cpf] Invalid CPF...' }

// Dígito verificador errado
handlers.cpf.safe('11144477700')
// → { valid: false, error: '[normalize:cpf] Invalid CPF...' }

// CPF válido
handlers.cpf.safe('11144477735')
// → { valid: true, value: '111.444.777-35', error: null }

Plugin: cnpj #

Valida pelo algoritmo de dígito verificador do CNPJ e formata no padrão 00.000.000/0000-00. Aceita CNPJ com ou sem máscara na entrada.

cnpj pt-BR 
// Apenas dígitos
handlers.cnpj.normalize('11222333000181')
// → '11.222.333/0001-81'

// Idempotente
handlers.cnpj.normalize('11.222.333/0001-81')
// → '11.222.333/0001-81'

// Sequência repetida rejeitada
handlers.cnpj.safe('00000000000000')
// → { valid: false, error: '[normalize:cnpj] Invalid CNPJ...' }

Plugin: cep #

Valida comprimento (exatamente 8 dígitos) e formata no padrão 00000-000. Não faz consulta de endereço — apenas valida o formato e aplica a máscara.

cep pt-BR 
// Formata a partir de dígitos
handlers.cep.normalize('01310100')
// → '01310-100'

// Idempotente
handlers.cep.normalize('01310-100')
// → '01310-100'

// Menos de 8 dígitos é inválido
handlers.cep.safe('0131010')
// → { valid: false, error: '[normalize:cep] CEP must have 8 digits' }

// Mais de 8 dígitos também é inválido
handlers.cep.safe('013101000')
// → { valid: false, error: '[normalize:cep] CEP must have 8 digits' }

Plugin: phone #

Suporta telefone fixo (10 dígitos: DDD + 8) e celular (11 dígitos: DDD + 9 + 8). O formato de saída detecta automaticamente o tipo e aplica a máscara correta.

phone pt-BR 
// Celular — 11 dígitos (DDD + 9 + número)
handlers.phone.normalize('11987654321')
// → '(11) 98765-4321'

// Fixo — 10 dígitos (DDD + número)
handlers.phone.normalize('1134567890')
// → '(11) 3456-7890'

// Idempotente
handlers.phone.normalize('(11) 98765-4321')
// → '(11) 98765-4321'

// Quantidade de dígitos incorreta
handlers.phone.safe('123')
// → { valid: false, error: '[normalize:phone] Expected 10 or 11-digit phone number' }

Plugin: rg #

Formata no padrão SP 00.000.000-X. Suporta RGs de 7 a 9 dígitos e o dígito verificador especial X. A opção format: 'digits' retorna apenas os dígitos sem máscara.

rg pt-BR 
// Padrão com 9 dígitos
handlers.rg.normalize('123456789')
// → '12.345.678-9'

// Dígito verificador X — aceito normalmente
handlers.rg.normalize('12345678X')
// → '12.345.678-X'

// Saída só com dígitos (sem máscara)
handlers.rg.normalize('123456789', { format: 'digits' })
// → '123456789'

Plugin: slug #

Converte qualquer string em slug seguro para URL — remove acentos, converte para lowercase, substitui espaços por hífens e remove caracteres especiais. Útil para gerar URLs de posts, IDs legíveis, nomes de arquivos, etc.

slug 
handlers.slug.normalize('Olá Mundo Legal!')
// → 'ola-mundo-legal'

handlers.slug.normalize('Café & Chá')
// → 'cafe-cha'

handlers.slug.normalize('  espaços   duplos  ')
// → 'espacos-duplos'

// Separador customizado
handlers.slug.normalize('Meu Post Incrível', { separator: '_' })
// → 'meu_post_incrivel'

Plugin: email #

Normaliza (trim + lowercase) e valida via RFC 5322 simplificado. Exige o formato local@dominio.tld — domínio sem ponto é rejeitado.

email 
// Normaliza espaços e capitalização
handlers.email.normalize('  JOAO@EMAIL.COM  ')
// → 'joao@email.com'

handlers.email.normalize('USUARIO@EMAIL.COM')
// → 'usuario@email.com'

// Formatos inválidos
handlers.email.safe('invalido')
// → { valid: false, error: '[normalize:email] Invalid email format' }

handlers.email.safe('sem@dominio')
// → { valid: false, error: '[normalize:email] Invalid email format' }

handlers.email.safe('ok@email.com')
// → { valid: true, value: 'ok@email.com', error: null }

Plugin: color #

Aceita #rgb, #rrggbb e rgb(r, g, b). Normaliza para lowercase hex por padrão. A opção format controla o formato de saída — útil para converter entre representações.

color 
// Shorthand hex expandido e lowercase
handlers.color.normalize('#abc')
// → '#aabbcc'

handlers.color.normalize('#FF0080')
// → '#ff0080'

// RGB → hex
handlers.color.normalize('rgb(255, 0, 128)')
// → '#ff0080'

// Formatos de saída
handlers.color.normalize('#ff0080', { format: 'rgb'       })
// → 'rgb(255, 0, 128)'

handlers.color.normalize('#ff0080', { format: 'hex-upper' })
// → '#FF0080'
